## Adjustment Rules
- If stars do not move but comments are active, tighten the CTA and move the repo ask later.
- If a claim triggers skepticism twice, the next post must add a stronger proof unit, not a stronger opinion.
- If readers ask the same scope question repeatedly, elevate that scope boundary into the next headline or first paragraph.
- If a post performs in Elixir circles but not broader webdev, rewrite the problem framing before reusing the proof.
- Keep the loop lightweight: manual observation is fine; do not block publishing on new analytics setup.

## Review Rhythm
- Run this scorecard once per week.
- Use the objection log to shape the next post hook, proof selection, and CTA order.
- Treat stars as the primary outcome and comments as diagnostic input.
